## File Purposes

### Data Files (Generated During Project)

**data/raw/**
- `race_results.csv` - All race results from 2014-2024 (season, round, driver, position, points, etc.)
- `qualifying_results.csv` - Qualifying positions and times (Q1, Q2, Q3)
- `circuits.csv` - Circuit metadata (name, country, coordinates)

**data/processed/**
- `merged_race_data.csv` - Race results merged with qualifying data
- `clean_race_results.csv` - Race data after cleaning and handling missing values
- `driver_performance.csv` - Driver statistics aggregated by season/circuit type

**data/features/**
- `track_features.csv` - Engineered track characteristics (speed, corners, elevation)
- `driver_features.csv` - Rolling averages, form metrics, circuit-type performance
- `team_features.csv` - Constructor strength metrics, development trajectory

### Notebooks

**01_data_collection.ipynb** ✅ READY
- Fetch data from Ergast F1 API
- Save raw data to CSV files
- Validate data completeness

**02_eda.ipynb** (Next to create)
- Visualize race results distributions
- Analyze driver/team performance trends
- Identify correlations between qualifying and race results
- Create initial insights for feature engineering

**03_feature_engineering.ipynb** 
- Create track characteristic features
- Build driver/team performance metrics
- Engineer rolling averages and form indicators
- Perform PCA on track features

**04_modeling.ipynb**
- Train baseline models
- Develop Ridge/Lasso regression models
- Build Random Forest and XGBoost models
- Compare model performance
- Feature importance analysis

**05_simulation.ipynb**
- Monte Carlo race simulation
- Probability distributions for outcomes
- Sensitivity analysis
- Generate predictions with confidence intervals

## Development Workflow

### Phase 1: Setup (Week 1)
1. Install dependencies: `pip install -r requirements.txt`
2. Run `01_data_collection.ipynb`
3. Verify data files in `data/raw/`

### Phase 2: EDA (Week 1-2)
1. Create `02_eda.ipynb`
2. Analyze distributions and patterns
3. Document findings in notebook

### Phase 3: Feature Engineering (Week 2-3)
1. Research track characteristics
2. Create `03_feature_engineering.ipynb`
3. Implement `feature_engineering.py`
4. Save features to `data/features/`

### Phase 4: Modeling (Week 3-4)
1. Create `04_modeling.ipynb`
2. Implement `models.py`
3. Train and evaluate models
4. Save best models to `outputs/models/`

### Phase 5: Simulation (Week 4-5)
1. Create `05_simulation.ipynb`
2. Implement `simulation.py`
3. Run Monte Carlo simulations
4. Generate probability distributions

### Phase 6: Finalization (Week 5-6)
1. Create dashboard (optional)
2. Write documentation in `docs/`
3. Polish README and presentation materials
4. Prepare portfolio deliverables
